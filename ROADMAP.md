# Roadmap

> Last updated: April 2026

This roadmap reflects our current plans. Priorities may shift based on community feedback and usage patterns.

## Q2 2026

- **Vector Embeddings for Memory** — Integrate embedding providers into memory recall for semantic similarity search (infrastructure exists, wiring in progress)
- **Enhanced A2A Protocol** — Add streaming, authentication, and task cancellation to the A2A v0.3 implementation
- **Python SDK** — Thin HTTP wrapper (`pip install qualixar-os`) for Python developers to interact with QOS
- **Expanded Built-in Tools** — Add http_request, json_transform, code_execute, database_query, and more
- **Programmatic API** — Library-mode exports for embedding QOS in TypeScript applications

## Q3 2026

- **Visual Workflow Builder** — Enhanced drag-and-drop agent composition in the dashboard
- **Cloud/Hosted Offering** — Managed QOS instance for teams without local infrastructure
- **Observability Dashboard** — Trace visualization, evaluation datasets, and prompt playground
- **Browser Automation** — Agent capability for web interaction and testing
- **SKILL.md Standard** — Marketplace compatibility with the emerging skill standard

## Q4 2026

- **Multi-Language SDKs** — Go and Java client libraries
- **gRPC Transport** — A2A v0.3 gRPC support alongside HTTP
- **Kubernetes Operator** — Declarative QOS deployment on Kubernetes
- **Self-Evolving Skills** — Automated skill mutation, blind verification, and evolution tracking

## How to Influence the Roadmap

Open a [Feature Request](https://github.com/qualixar/qualixar-os/issues/new?template=feature_request.md) or start a Discussion. Community demand directly influences prioritization.
