# Governance

## Decision-Making Model

Qualixar OS uses a **BDFL (Benevolent Dictator for Life)** governance model during its early growth phase.

**Lead Maintainer:** Varun Pratap Bhardwaj ([@varun369](https://github.com/varun369))

## How Decisions Are Made

### Minor Changes (bug fixes, docs, small features)
- Open a Pull Request with a clear description
- Maintainer reviews within 7 days
- Merge on approval

### Major Changes (new modules, breaking changes, architecture shifts)
- Open a Discussion or Issue tagged `[RFC]` first
- Community has 14 days to comment
- Maintainer makes the final decision after considering feedback
- Decision is documented in the issue before implementation begins

## Release Cadence

- **Patch releases:** As needed for bug fixes
- **Minor releases:** Monthly for new features
- **Major releases:** Semantic versioning, announced 30 days in advance
- All releases follow [Semantic Versioning](https://semver.org/)
